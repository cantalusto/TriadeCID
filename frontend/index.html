CONFIDENCIALIDADE

Não vazar segredos em respostas de erro

O endpoint /api/leak-demo lança de propósito um erro contendo API_KEY, senha de banco e JWT secret. O middleware de erros no servidor redige tudo antes de responder.

  • No console do servidor você verá o log completo (uso interno).
  • No navegador, abaixo, você verá apenas uma mensagem genérica.

INTEGRIDADE

Preço calculado no servidor (nunca confie no Front-End)

O navegador envia um carrinho em que tenta pagar R$ 0,03 por um Notebook + 2 Mouses. O servidor ignora o total e os price enviados e recalcula tudo pelo banco de dados. 

—
DISPONIBILIDADE

Balanceamento de carga com Node cluster

/api/products/report simula uma operação pesada de CPU. Disparamos várias em paralelo: o cluster distribui entre workers (um por núcleo). Veja múltiplos PIDs sendo usados — evidência do balanceamento. 

—
JWT — AUTENTICAÇÃO

Autenticação por crachá digital

Faça login pela rota segura, guarde o token, e use-o para acessar rotas protegidas. O token tem 3 partes (Header.Payload.Signature); qualquer alteração no payload quebra a assinatura.

UX DEFENSIVA

Front-End como primeira camada de defesa

Pequenos cuidados de interface que reduzem a superfície de ataque e melhoram a UX: máscara de input, popup de confirmação, botão desabilitado após clique, loading visível.
